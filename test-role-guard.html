Prueba de Race Condition Fix en RoleGuard

Problema Original

El useEffect tenía hasAccess en el array de dependencias y se usaba inmediatamente después de setHasAccess(access), causando race condition.

// CÓDIGO ANTIGUO (con race condition) useEffect(() => { const checkAuthAndRole = () => { // ... cálculo de access setHasAccess(access); setIsChecking(false); // ⚠️ Usaba hasAccess (state) inmediatamente después de setHasAccess if (requireAuth && authStatus && !hasAccess) { router.push("/unauthorized"); } }; checkAuthAndRole(); }, [..., hasAccess]); // ⚠️ hasAccess en dependencias

Solución Implementada

Ahora se usa una variable local access para decisiones de redirección y se eliminó hasAccess de las dependencias.

// CÓDIGO NUEVO (sin race condition) useEffect(() => { const checkAuthAndRole = () => { let access = true; // ... cálculo de access usando variable local setHasAccess(access); // Solo para UI setIsChecking(false); // ✅ Usa variable local 'access' para decisiones if (requireAuth && authStatus && !access) { router.push("/unauthorized"); } }; checkAuthAndRole(); }, [..., /* sin hasAccess */]); // ✅ hasAccess removido

Cambios clave:

  1. Variable local access para decisiones de redirección
  2. hasAccess removido del array de dependencias
  3. Redirecciones basadas en variable local, no en state
  4. Mantiene normalización de roles con toUpperCase()
